CI/CD: Versioning and K8s Deploy

Contents

  • get-version.sh – Used by pipelines to compute version (from git tag or 1.0.0-YYYYMMDD.BuildId).
  • azure-pipelines.build-deploy.yml – Template pipeline: build image with version, push, deploy to K8s.

Quick setup in Azure DevOps

1. Registry

  • Create a service connection for your container registry (Docker Hub or Azure ACR).
  • Ensure the pipeline has permission to use it.

2. Kubernetes

  • Create a Kubernetes service connection (kubeconfig or AKS).
  • Create an Environment (e.g. k8s-leet-monitor) and add the Kubernetes resource so the deploy job can run kubectl.

3. Pipeline

  • In the repo (e.g. vendon-sync synced to Azure DevOps), add a pipeline that uses the template, or copy azure-pipelines.build-deploy.yml into the repo root as azure-pipelines.yml.
  • Set parameters (or variables):
    • imageName: e.g. programmeradmin25/vendon-sales-api
    • dockerfilePath: e.g. Dockerfile.api
    • k8sDeployment: e.g. vendon-sales-api
    • k8sNamespace: e.g. leet-monitor
    • containerName: e.g. api

4. Versioning behaviour

  • Push to main: version = 1.0.0-YYYYMMDD.BuildId, image tagged with that and latest.
  • Push tag v1.2.3: version = 1.2.3, image tagged with that (and optionally latest).

Deploy stage runs after a successful build and updates the deployment with the new image tag.

Example: vendon-sync

In Azure DevOps (Leet Motion or the project where vendon-app is synced), create a pipeline:

  • Pipeline YAML: point to the template or use:
# azure-pipelines.yml (in vendon-app repo)
trigger:
  - main
resources:
  repositories:
    - repository: self
parameters:
  - name: imageName
    value: programmeradmin25/vendon-sales-api
  - name: dockerfilePath
    value: Dockerfile.api
  - name: k8sDeployment
    value: vendon-sales-api
  - name: k8sNamespace
    value: leet-monitor
  - name: containerName
    value: api
# Then extend the template or copy stages from azure-pipelines.build-deploy.yml

If the template lives in another repo, use extends or copy the template into this repo and reference it.
